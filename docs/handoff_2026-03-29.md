# 开发交接文档（2026-03-29）

## 1. 当前目标

项目当前处于第一阶段：基于 `ESP-Brookesia` 为 `ESP32-S3 Touch AMOLED 2.06` 做一个从 0 开始的 `Settings` App。

当前已实现能力：

- Wi-Fi 扫描
- Wi-Fi 连接
- 屏幕亮度调节
- 音量调节
- 自动对时 / 手动设时
- 设备信息查询
- 电池电量、充电状态、PMU 温度
- 电池电压、系统电压
- Device Info 折叠/展开

## 2. 当前状态

截至 2026-03-29 21:41:53 +08:00：

- 代码已完成构建
- 最新固件已成功刷入 `COM7`
- Settings App 已能在启动器中显示
- `Device Info` 当前默认收起，点击头部可展开/收起

## 3. 关键文件

核心实现文件：

- `docs/software_design.md`
- `components/settings_service/settings_service.hpp`
- `components/settings_service/settings_service.cpp`
- `components/brookesia_app_settings/esp_brookesia_app_settings.cpp`
- `components/brookesia_app_settings/CMakeLists.txt`
- `components/storybook_axp2101/CMakeLists.txt`
- `main/main.cpp`
- `.gitignore`

关键点说明：

- `settings_service` 负责 Wi-Fi / 时间 / 亮度 / 音量 / PMU 状态读取
- `brookesia_app_settings` 只负责设置页 UI 和交互
- `storybook_axp2101` 是从 Waveshare `01_AXP2101` 例程裁剪出来的最小 PMU 组件
- `main/main.cpp` 里会刷新状态栏时钟、Wi-Fi、电池图标
- `components/brookesia_app_settings/CMakeLists.txt` 使用了 `WHOLE_ARCHIVE`，否则 Settings 图标可能不注册

## 4. 已知设计结论

- 不参考旧 `07` 工程做重构，当前方案是从 0 开始的新架构
- `Device Info` 没有单独拆成新 App，而是保留在 `Settings` 里
- 当前显示的温度是 `AXP2101 PMU` 内部温度，不是电池 NTC 温度
- PMU 读取来自板载 AXP2101，当前已启用电池电压、系统电压、VBUS、电池百分比、充电状态、温度读取

## 5. 当前已解决的问题

已修复过的典型问题：

- 启动器主页空白，没有 App 图标
- Wi-Fi 密码输入框和键盘不同屏
- Device Info 内容过高，默认占位太大
- 设备信息缺少电池、充电状态、温度
- 设备信息缺少电压

## 6. 下一步建议

优先级建议如下：

1. 真机回归验证
2. 校验 Wi-Fi 错误路径和重连体验
3. 评估是否需要增加独立的电源详情页
4. 整理提交范围并提交 git

建议优先真机验证这些点：

- `Device Info` 默认是否收起
- 点按头部后是否能展开/收起
- `Battery Volt` / `System Volt` 是否显示合理值
- 状态栏电池图标是否会随充电状态变化
- 重启后亮度 / 音量 / Wi-Fi 凭据是否持久化
